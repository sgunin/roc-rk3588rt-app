Error Handling and Error Code

After being called, API of the SDK will return the integer value that indicates the error code. The status code is hexadecimal with the format “0x%x”.
If the API calling is normal, and there is no error detected, return “0x00000000” that indicates “no error”, otherwise the error code will be returned.
Note
Go to file “MvErrorDefine.h” or Error Code for the error code, its definition and explanation.
You can refer to the following instance, check the API exception and return error.
int nRet = MV_OK;
nRet = MV_CC_OpenDevice(handle);
if (MV_OK != nRet)
{
printf("Open Device fail! nRet [0x%x]\n", nRet);
return nRet;
}
The instance shows the status of getting the calling of MV_CC_OpenDevice() when opening the camera.
  • If the camera can be opened normally, this instance will not return error.

  • If the camera is connected to the third-party software when calling API, MV_CC_OpenDevice() fails to open the camera normally. In this case, the API will return following information.

    Open Device fail! nRet [0x80000203]

    After getting error information and status code, go to Error Code for details. In this instance, “0x80000203” represents “no access permission”.
