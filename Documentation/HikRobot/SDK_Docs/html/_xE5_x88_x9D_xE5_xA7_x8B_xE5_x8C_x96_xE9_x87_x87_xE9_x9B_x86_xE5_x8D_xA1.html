Frame Grabber Initialization

This chapter introduces how to call the SDK APIs to control frame grabbers, such as enumeration, opening/closing, configuring frame grabber.

Basic flows include:
  1. Enumerate Frame Grabbers
  2. Create Frame Grabber Instance
  3. Open Frame Grabber
  4. (Optional) Configure Parameters
  5. Close Frame Grabber
  6. Destroy Frame Grabber Instance

Prerequisite

Before you continue, make sure SDK has been initialized. For more details, refer to SDK Initialization.

Step 1 Enumerate Frame Grabber(s)

Call MV_CC_EnumInterfaces() and enter the frame grabber type nTLayerType, to get the frame grabber list of the corresponding type (pInterfaceInfoList).
The example shows the enumeration of CoaXPress and XoFLink frame grabber.
MV_INTERFACE_INFO_LIST stInterfaceInfoList={0};
memset(&stDeviceList, 0, sizeof(MV_CC_DEVICE_INFO_LIST));
nRet = MV_CC_EnumInterfaces(MV_CXP_INTERFACE|MV_XOF_INTERFACE, &stInterfaceInfoList); //Configure “MV_CXP_INTERFACE” and “MV_XOF_INTERFACE” in nTLayerType
Check(nRet);
Note
Details for frame grabber interfaces:
Definitions of Frame Grabber Interfaces Value Description
MV_GIGE_INTERFACE 0x00000001 GigE Vision Frame Grabber
MV_CAMERALINK_INTERFACE 0x00000004 Camera Link Frame Grabber
MV_CXP_INTERFACE 0x00000008 CoaXPress Frame Grabber
MV_XOF_INTERFACE 0x00000010 XoFLink Frame Grabber

Step 2 Create Frame Grabber Instance

You can create frame grabber instance through frame grabber information or ID.

  • Option 1: Call MV_CC_CreateInterface() and enter the frame grabber information pInterfaceInfo to get the frame grabber instance. Different frame grabber instances will be created according to frame grabber information.

    void* hInterface = NULL;
    // Create a frame grabber handle
    nRet = MV_CC_CreateInterface(&hInterface, stInterfaceInfoList.pInterfaceInfos[nIndex]);
    Check(nRet);

  • Option 2: Call MV_CC_CreateInterfaceByID() and enter the frame grabber ID pInterfaceID to get the frame grabber instance, ID of which can be the serial No.

    Note
    In this way, do not enumerate frame grabber in advance, configure the serial No. of frame grabber.

    The following code shows how to enumerate the frame grabber with the serial No. “DA1433238” through MV_CC_CreateInterfaceByID().

    void* hInterface = NULL;
    const char *pInterfaceID = "DA1433238";
    // Create a frame grabber handle
    nRet = MV_CC_CreateInterfaceByID(&hInterface, pInterfaceID);
    Check(nRet);

Step 3 Open Frame Grabber

"Open frame grabber" refers to establishing the connection between the frame grabber instance and physical frame grabber, and making the frame grabber instance gaining access to the frame grabber.
Open frame grabber through calling MV_CC_OpenInterface(). The return value MV_OK stands for an opening success. Other return value indicates an opening failure.
// Open the frame grabber
nRet = MV_CC_OpenInterface(hInterface, NULL);
Check(nRet);


(Optional) Step 4 Configure Parameters

Corresponding SDK APIs can be called after opening frame grabber to achieve functions as follows:
  • Realizing parameters configuration of frame grabber. For more details, refer to Parameter Configuration.
  • Getting frame grabber events. For more details, refer to Event and Exception.

Step 5 Close Frame Grabber

When the communication between frame grabber instance and physical frame grabber is no longer needed, call MV_CC_CloseInterface() to close the frame grabber.
The sample code is as follows:
// Close the device.
nRet = MV_CC_CloseInterface(hInterface);
Check(nRet);


Step 6 Destroy Frame Grabber Instance

Call MV_CC_DestroyInterface() to destroy frame grabber instance and release the resource occupied by the instance.
Attention
After destroying frame grabber instance, the previous instance will be invalid, and cannot be used for other APIs.
The sample code is as follows:
// Destroy the handle
nRet = MV_CC_DestroyInterface(hInterface);
Check(nRet);
Note
After realizing functions through SDK, you need to deinitialize SDK to release system resource occupied by SDK. For more details, refer to SDK Initialization.
