Update History
 
Version Description
Version 4.4.0_June/2024 Added an API for importing device attribute: MV_CC_FeatureLoadEx().
Added data structures: MVCC_NODE_ERROR for node error and MVCC_NODE_ERROR_LIST for node error list.
Added sample code for quick software trigger: QuickSoftwareTrigger.cpp.
Updated the name of sample code for acquiring image via callback to GrabImage_Callback.cpp.
Updated console sample code for Linux operating system: InterfaceDemo.cpp and ReconnectDemo.cpp.
Updated Status Code.
Moved Status Code to API Interface.
Version 4.3.0_December/2023 Added APIs of SDK Initialization: initializing SDK MV_CC_Initialize() and finalizing SDK MV_CC_Finalize().
Added APIs of Frame Grabber Initialization: enumerating frame grabbers MV_CC_EnumInterfaces(), and creating frame grabber handle MV_CC_CreateInterface() and MV_CC_CreateInterfaceByID(), opening frame grabber MV_CC_OpenInterface(), closing frame grabber MV_CC_CloseInterface(), and destroying frame grabber handle MV_CC_DestroyInterface().
Added APIs of CameraLink Camera: getting serial port information list MV_CAML_GetSerialPortList() and setting specified enumeration list of serial ports MV_CAML_SetEnumSerialPorts().
For APIs of Image Rendering, supports rendering images of width (max. 16384) and height (max. 163840).
For the API MV_CC_SetBayerCvtQuality() in Image Processing, extended the parameter BayerCvtQuality by adding the value of optimal plus.
For APIs of Image Processing and Attribute Configuration, supports frame grabber handle.
For APIs of Device Upgrade, supports frame grabber firmware upgrade.
For APIs of Event and Exception, supports frame grabber events.
Added structures of frame grabber information: MV_INTERFACE_INFO, and of frame grabber information list: MV_INTERFACE_INFO_LIST.
Added structures of serial port information: MV_CAML_SERIAL_PORT, and of serial port information list: MV_CAML_SERIAL_PORT_LIST.
Added structures of CoaXPress camera information: MV_CXP_DEVICE_INFO, of CameraLink camera information list: MV_CML_DEVICE_INFO, and of XoFLink camera information: MV_XOF_DEVICE_INFO.
Added console sample code for Linux operating system: Display.cpp, ForceIPEx.cpp, Grab_Asynchronous.cpp, InterfaceAndDeviceDemo.cpp, and InterfaceDemo.cpp.
Updated the structures of saving images to files: MV_SAVE_IMAGE_TO_FILE_PARAM_EX, and of image saving: MV_SAVE_IMAGE_PARAM_EX3.
Updated the structures of displayed frame information: MV_DISPLAY_FRAME_INFO and MV_DISPLAY_FRAME_INFO_EX.
Updated the structures of CCM parameters: MV_CC_CCM_PARAM and MV_CC_CCM_PARAM_EX.
Updated the structure of device information: MV_CC_DEVICE_INFO.
Updated the sharpness structure: MV_CC_SHARPEN_PARAM.
Updated Error Code: added MV_E_KEY_VERIFICATION.
Adjusted the structure of API Reference.
V4.1.0 Linux_March 2023
All APIs supported being used in multiple systems (Windows32/64, Linux32/64, armhf, aarch64, arm-none). The SDK function modules became consistent in Linux and Windows.
Supported controlling camera demo (BasicDemoByGenTL) and line camera demo (BasicDemoLineScan) by loading CTI files.
Added lossless decoding sample program: HighBandwithDecode.cpp
Added API for saving images: MV_CC_SaveImageEx3() , and obsoleted the old API: MV_CC_SaveImageEx2() .
Added API for saving images to files: MV_CC_SaveImageToFileEx(), and obsoleted the old API: MV_CC_SaveImageToFile() .
Added API for converting pixel format: MV_CC_ConvertPixelTypeEx(), and obsoleted the old API: MV_CC_ConvertPixelType() .
Added structure of image saving parameters: MV_SAVE_IMAGE_PARAM_EX3 .
Added structure of saving images to files parameters: MV_SAVE_IMAGE_TO_FILE_PARAM_EX .
Added structure of pixel format conversion parameters: MV_CC_PIXEL_CONVERT_PARAM_EX .
Added driver installation steps in Integrated Development Environment .
Added some error codes.
