Sample Program Overview

After successful installation of the client, you can get files of sample programs under the installation path of the client. Based on these provided sample programs, corresponding functions can be achieved in your programs.
Note
For example, for 64-bit operating system, the sample programs are under the path of /Samples/64/C++/.
In the folder of Samples, the sample programs categories are as follows.
  • Basic Sample Programs
  • Frame Grabber Sample Programs
  • Area Scan Camera Sample Programs
  • Line Scan Camera Sample Programs

Basic Sample Programs

You can get basic sample programs under the path of ..\Development\Samples\General. The basic sample programs, including camera connection, image acquisition, image processing, parameter configuration, and events, are as follows.
Basic Sample Programs of Camera Connection
File Name Sample Program Description
ConnectSpecCamera.cpp Connect to Camera by IP Address It shows how to connect camera to your program by camera IP address and NIC IP address corresponding to the camera.
Note
Enter IP address in the format of xx.xx.xx.xx.
ForceIPEx.cpp Force Set IP Address for GigE Cameras It shows how to force set IP address for GigE cameras to achieve camera reconnection and camera re-enumeration in power-off situations.
ReconnectDemo.cpp Reconnect to Cameras It shows how to reconnect to a camera within the heartbeat time when the camera goes offline.


Basic Sample Programs of Image Acquisition
File Name Sample Program Description
Grab_Asynchronous.cpp Get and Process Image(s) in Asynchronous Mode It shows how to get and process images in asynchronous mode.
Note
It prevents the long time cost issues caused by synchronous operations.
GrabImage.cpp Get Image(s) via Polling It shows how to get images via polling function.
GrabImage_Callback.cpp Get Image(s) via Callback Function It shows how to get images via callback function.
GrabImage_HighPerformance.cpp Get Image(s) in High-Performance Method It shows how to get one or more images in high-performance method.
GrabMultipleCamera.cpp Get Image(s) from Multiple Camera(s) It shows how to get one or more images from multiple cameras. You can create multiple camera handles to distinguish different cameras.


Basic Sample Programs of Image Processing
File Name Sample Program Description
ChunkData.cpp Get Chunk Data It shows how to enable Chunk Data, configure Chunk Data parameters, and get Chunk Data. Chunk Data examples: Timestamp and Exposure.
Display.cpp Display Camera Image(s) It shows how to call related APIs to display images acquired by cameras.
HighBandwidthDecode.cpp High-Bandwidth Lossless Compression It shows how to decode lossless compression stream from the camera into raw data via high-bandwidth lossless compression related algorithm.
ImageProcess.cpp Process Images It shows how to convert image pixel format and save images.
ImageSave.cpp Save Image(s) from Camera(s) It shows how to save one or more images from cameras in RAW, JPEG, BMP, TIFF, and PNG format.


Basic Sample Programs of Parameter Configuration
File Name Sample Program Description
MultiCast.cpp Set Multicast Mode It shows how to set transmission mode to multicast mode, that is, querying the user to launch the multicast controlling application or the multicast monitoring application, and opening camera and setting multicast mode in the specified access permission.
ParametrizeCamera_FileAccess.cpp Import File(s) to / Export File(s) from Camera(s) It shows how to import file(s) to / export file(s) from camera(s), and get file accessing progress.
ParametrizeCamera_LoadAndSave.cpp Import/Export Camera Attribute File(s) It shows how to export camera attribute tree settings to .ini files and import .ini files to camera attribute tree settings.


Basic Sample Programs of Events
File Name Sample Program Description
Events.cpp Get Camera Events It shows how to configure camera event features, register an event callback, and process the events acquired from the event callback. Event example: ExposureEnd.


Sample Programs of Frame Grabber(s)

You can get basic sample programs under the path of ..\Development\Samples\FrameGrabber.
File Name Sample Program Description
Events_Interface.cpp Open and Receive Frame Grabber Event(s) It shows how to open and receive one or more frame grabber events.
InterfaceAndDeviceDemo.cpp Control Camera(s) Connected to Frame Grabber(s) It shows how to enumerate one or more cameras connected to frame grabber(s) via enumeration APIs, and perform operations such as opening cameras and grabbing images.
InterfaceDemo.cpp Enumerate and Control Frame Grabber(s) It shows how to enumerate one or more frame grabbers via enumeration APIs and perform operations such as opening frame grabbers and setting attributes.
QuickSoftwareTrigger.cpp Achieve Software Trigger of Frame Grabbers It shows how to achieve quick software trigger of some frame grabbers. For supported frame grabber models, refer to actual conditions.


Sample Programs of Area Scan Camera(s)

You can get basic sample programs under the path of ..\Development\Samples\AreaScanCamera.
File Name Sample Program Description
ParametrizeCamera_AreaScanIOSettings.cpp Configure Trigger Node IO Settings of Area Scan Cameras It shows how to configure trigger node IO settings of area scan cameras.
SetParam.cpp Set Camera Parameters It shows how to use general APIs to set camera parameters, including settings of integer, float, enumeration, and boolean nodes.
Trigger_Image.cpp Get Camera Image(s) in Trigger Mode It shows how to get images in polling function via software trigger mode.
Steps are as follows: select a camera and create a handle, open the camera, set trigger mode, set trigger source (software trigger), start grabbing images, create thread for image grabbing (send software trigger command and call MV_CC_GetOneFrameTimeout() to get images), end grabbing images, close the camera, and destroy the camera handle.
Trigger_ImageCallback.cpp Trigger_ImageCallback It shows how to get images in callback function via software trigger mode. Steps are as follows: select a camera and create a handle, open the camera, set trigger mode, set trigger source (software trigger), register an callback function for image grabbing, start grabbing images, get image information in callback function via software trigger mode, end grabbing images, close the camera, and destroy the camera handle.


Sample Programs of Line Scan Camera(s)

You can get basic sample programs under the path of ..\Development\Samples\LineScanCamera.
File Name Sample Program Description
MultiLightCtrl_ImageStitching.cpp Multi-Light Control, Image Division, and Image Stitching of Line Scan Cameras It shows how to set multi-light control parameters (the default is 2), divide the image and stitch the divided images in top-bottom direction to create a new image, and save the new image to local in BMP format.
ParametrizeCamera_LineScanIOSettings.cpp Configure IO Settings of Line Scan Cameras It shows how to configure node IO settings of frame trigger and line trigger for line scan cameras by calling MV_CC_SetEnumValue().
